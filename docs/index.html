👉 近年來「以使用者為中心」的概念已深植人心， 而這股力量也開始擴散至無障礙網頁設計領域， 藉由不斷改善網頁的親和力， 期望能達成通用設計的核心概念：Design for All。

👉 Piman 正是為此而生， 從人們日常生活常頻繁接觸的網頁開始著手改善， 所以首要任務便是推廣開發者使用 Piman 開發政府機關網站， 進而應用在產品、官網等地方，如此企業可善盡社會責任，也可與國際趨勢接軌。

應該使用符合無障礙網頁設計 UI 框架的其中 7 個理由
💪💪💪
  • 不可能要求所有參與人員都學習無障礙網頁設計規範
    我們鼓勵越多人認識越好，但這是不切實際的。使用 Piman 就省下很多人的力氣與成本。
  • 每次都要處理很多額外的東西
    例如：可點擊的地方無法使用鍵盤操作。
  • 使用的前端框架不符合無障礙網頁設計規範
    可能必須修改原始碼，造成鎖死版本問題，亦或是額外使用暴力法...。
  • 每次機器掃描後心都很累（臺灣 NCC freego 軟體）
    團隊內如果沒有人特別學習，就會出現改得很煩、不知道解法、團隊氣氛糟糟的情況。
  • 很大機率被退件 （臺灣 NCC 人工審核）
    增加專案被罰錢的風險（ 最好能談成維運時再通過無障礙標章，也比較合理 ）。
  • 沒有認真研究的話，有需要的人會感到困擾
    原本是好意，結果使用起來變得更困擾。
  • 公司都應該需要一套 Design System
    既然如此，那何不順便讓網頁親和力變得更高呢！
基於 Vue 2 設計
  • 因為無障礙網頁設計有許多規範，所以Piman 使用前端框架 Vue 製作各種元件，讓開發者可以專注在優先度更高的事情上，不需因為無障礙網頁設計而煩惱。
  • 選擇了 Vue 也相對其他框架好入門。
  • 為了良好的支援 IE11 ，選擇了 Vue 2。

FAQ

  • 您必須熟讀 HTML 字典 與  CSS 字典

  • 推薦至無障礙網路空間服務網熟讀各項無障礙網頁設計規範

  • 您需要了解 WAI-ARIA 並謹記 WAI-ARIA 的宗旨為：No ARIA is better than Bad ARIA，這也意味著熟讀 HTML 字典與 CSS 字典。大概先暸解這三個，之後可以再慢慢擴充相關知識。

現代主流瀏覽器與 IE11+

當然有，只要您想要設計無障礙網頁，我們很樂意提供您除了 Piman 之外的資訊。

目前主流 UI 框架應該都有符合無障礙網頁設計規範，例如 Bootstrap，而我們私心推薦的是 IBM 的 Carbon，因爲它包含了三大前端框架，且有完整的 demo 和 文件。 或是可以選擇類似例如：https://headlessui.dev/ 跟 https://www.radix-ui.com/ 等 headless，這些專注在行為上，UI 風格則是自己額外建立。

只不過有一點您可能要注意的是，因為 Piman 是有對臺灣 NCC 規範特別關注，再加上臺灣政府單位大多數仍有 IE11 的需求， 所以以上框架您可能需要自行將程式碼轉成 IE11 以及可能會碰到臺灣 NCC freego 機器掃描和人工檢核的問題。
